# 开发计划

## 架构总览

```
微信小程序
├── 页面
│   ├── 首页（每日卡片）
│   └── 设置页（出生信息 + 体质问卷）
├── 云函数（后端）
│   ├── baziCalc    —— 八字排盘
│   ├── emotionEngine —— 七层情绪规则
│   ├── weatherFetch —— Open-Meteo 天气
│   └── dailyCard   —— 组装完整卡片
└── 云数据库
    └── userProfile —— 出生信息 + 体质结果 + 城市
```

## Phase 1：八字排盘 · 云函数

**目标：** 输入出生时间+性别 → 输出完整八字盘

**内容：**
- 公历转农历
- 排四柱（年月日时干支）
- 排十神（日干vs各柱天干）
- 排大运（起运岁数+每十年干支）
- 排空亡（日柱所在旬）
- 用 ao031 八字（1997-03-19 18:30 男）验证结果

**产出：** `baziCalc` 云函数

**验证标准：** 输出结果 = 丁丑 癸卯 庚申 乙酉，庚子大运，甲寅旬空亡子丑

---

## Phase 2：情绪引擎 · 云函数

**目标：** 八字盘 + 当前日期 → 情绪描述 + 时辰建议 + 短句

**内容：** 实现情绪规则映射表（十二节）的七层规则：
1. 六十甲子日柱性情
2. 主十神 + 辅十神 + 旺衰
3. 刑冲合害（流日支 vs 八字四支）
4. 流月背景
5. 大运底色
6. 空亡判定
7. 情志五脏 + 季节加权

综合权重输出 → 大白话翻译 → 匹配短句

**产出：** `emotionEngine` 云函数

**验证标准：** 用 ao031 八字跑 2026-06-14，输出情绪描述和短句，人工审查是否合理

---

## Phase 3：天气 + 卡片组装

**目标：** 拿到天气数据，把天气+情绪组装成完整卡片

**内容：**
- `weatherFetch` 云函数：调 Open-Meteo，输入城市经纬度 → 输出当日气温、天气类型、降水
- `dailyCard` 云函数：调 emotionEngine + weatherFetch → 穿衣建议 + 饮食推荐 + 组装完整卡片
- 穿衣规则：气温区间 + 体质寒热 → 一句
- 推荐规则：季节 + 天气 + 体质 → 一句

**产出：** `weatherFetch` + `dailyCard` 两个云函数

---

## Phase 4：前端页面

### 4a. 首页 —— 每日卡片

**目标：** 打开小程序看到当天卡片，一屏到底

**内容：**
- 调用 `dailyCard` → 渲染卡片
- 日期+天气栏
- 情绪描述区域
- 时辰建议
- 穿衣+推荐
- 短句收尾（古文+翻译+拉回今天）
- 中国风极简设计（宣纸底色、传统字体，参考吾乡帖家书主题）
- 本地缓存：当天卡片缓存，离线也能看

### 4b. 设置页

**目标：** 输入/修改出生信息，可选做体质问卷

**内容：**
- 出生日期选择器 + 时间选择器 + 性别
- 城市选择器（省份→城市联动，复用吾乡帖 cityData）
- 体质问卷入口（28题，复用吾乡帖 quiz.js 的 constGroups）
- 保存到云数据库

---

## Phase 5：收尾

**内容：**
- 错误处理（网络异常、API超时兜底）
- 定时触发器：每天凌晨自动跑一次 dailyCard，结果缓存，用户早上打开秒加载
- ICP备案材料准备
- 提交审核上架
